# Potter for Zen Browser

A chrome theme for [Zen Browser](https://zen-browser.app), the calm Firefox fork.
Because Zen is Firefox underneath, Potter ships as a `userChrome.css` stylesheet
that recolors the toolbar, URL bar, tabs, sidebar, menus, and find bar. Ships all
three flavors: **Parchment** (light), **Quill** (dark, default), **Ink** (deepest
dark).

| File | What it is |
|------|------------|
| `potter-quill.css` / `potter-parchment.css` / `potter-ink.css` | `--potter-*` custom properties + remapped Zen/Firefox chrome variables and surface overrides. |
| `Apply-Potter-Zen.ps1` | One-shot installer: copies the flavor into your profile's `chrome/`, wires up `userChrome.css`, and enables the legacy-stylesheets pref. |

## Apply it (Windows, scripted)

From this folder:

```powershell
.\Apply-Potter-Zen.ps1 -Flavor quill
```

It targets the default profile from `%APPDATA%\zen\profiles.ini`. Use
`-Flavor parchment|ink` for another flavor, `-AllProfiles` to do every profile,
or `-ProfilePath "<full path>"` to pick one. Existing `userChrome.css` / `user.js`
are backed up as `*.potter.bak`. **Fully quit and reopen Zen** to see it.

## Apply it (manual, any OS)

1. Find your profile folder (`about:support` → *Profile Directory*) and create a
   `chrome/` subfolder if it doesn't exist.
2. Copy the flavor you want (e.g. `potter-quill.css`) into `chrome/`.
3. Create/edit `chrome/userChrome.css` and add:
   ```css
   @import "potter-quill.css";
   ```
4. In `about:config`, set
   `toolkit.legacyUserProfileCustomizations.stylesheets` to `true`.
5. Restart Zen.

Potter rides alongside Zen Mods — it touches `userChrome.css`, not the
mod-managed `chrome/zen-themes.css`. The accent is **peach**; reach for any
`--potter-*` variable to theme more of the chrome yourself.
