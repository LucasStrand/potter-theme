# Potter for Windows (desktop theme)

Sets Potter's accent color and light/dark mode system-wide on Windows 10/11 —
title bars, taskbar, Start, and the Settings accent swatch. Ships all three
flavors: **Parchment** (light), **Quill** (dark, default), **Ink** (deepest dark).

| File | What it is |
|------|------------|
| `Apply-Potter.ps1` | The way to apply Potter. Sets accent + dark/light in the registry and refreshes the shell. |
| `potter-quill.theme` / `potter-parchment.theme` / `potter-ink.theme` | Optional `.theme` entries for the Settings → Themes gallery. |

## Apply it

```powershell
# from this folder
./Apply-Potter.ps1                 # Quill  (default, dark)
./Apply-Potter.ps1 -Flavor parchment
./Apply-Potter.ps1 -Flavor ink
```

If PowerShell blocks the script, run it for this session only:

```powershell
powershell -ExecutionPolicy Bypass -File .\Apply-Potter.ps1 -Flavor quill
```

The accent and mode apply immediately (title bars/taskbar/Start); a few apps pick
up the change on your next sign-in. It only writes to `HKEY_CURRENT_USER` — no
admin rights, nothing machine-wide.

### Why a script and not just the `.theme`?

On Windows 10/11 a `.theme` file no longer drives the **live** accent color or
app/system dark mode — the OS reads those from the registry. Double-clicking a
`potter-*.theme` adds it to your Themes gallery but won't reliably recolor the
desktop, and on some builds throws an "couldn't apply theme" dialog. So
`Apply-Potter.ps1` writes the registry directly:

- `…\Themes\Personalize` → `AppsUseLightTheme` / `SystemUsesLightTheme` (dark for
  Quill/Ink, light for Parchment) + `ColorPrevalence`.
- `…\DWM` → `AccentColor` (ABGR) + `ColorizationColor` (ARGB) for title bars.
- `…\Explorer\Accent` → `AccentColorMenu` / `StartColorMenu` + an 8-shade
  `AccentPalette` ramp so Settings shows the right swatch and tints.
- Broadcasts `WM_SETTINGCHANGE` ("ImmersiveColorSet") so the shell repaints
  without a sign-out.

The accent is Potter `peach` (`#e08a6a` in Quill).

## Revert

Pick any stock theme under **Settings → Personalization → Themes** (e.g. "Windows
(dark)"), or re-run the script with a different flavor.
